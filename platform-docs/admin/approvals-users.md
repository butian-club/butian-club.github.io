# 账号审批与用户管理

新账号注册后不会立刻可用，需要管理员审批。审批通过的账号还要在用户管理里
分配角色与组别，才能正常参与协作。本页覆盖这两块管理侧操作。

- ✅ 审批地址：[https://gfssm.butian.club/admin/approvals](https://gfssm.butian.club/admin/approvals)
- 👥 用户地址：[https://gfssm.butian.club/admin/users](https://gfssm.butian.club/admin/users)

## 账号状态四态

平台账号是一个状态机，共四种状态：

| 状态 | 含义 | 能否登录 |
| --- | --- | --- |
| **PENDING** | 已注册，待审批 | 否 |
| **ACTIVE** | 已激活，正常使用 | 是 |
| **REJECTED** | 审批被拒绝 | 否 |
| **DISABLED** | 已被禁用 | 否 |

各状态之间的流转（谁能触发、能否恢复）详见
[账号状态说明](../reference/account-states.md)。

## 账号审批

`/admin/approvals` 列出所有**待审批（PENDING）**用户。对每一条，你可以：

- **通过**：账号变为 **ACTIVE**，用户即可登录并开始使用。
- **拒绝**：账号变为 **REJECTED**，用户无法登录。

### 审批操作步骤

1. 打开 [https://gfssm.butian.club/admin/approvals](https://gfssm.butian.club/admin/approvals)。
2. 查看列表中每个待审批用户的注册信息（用户名、注册时间等）。
3. 确认是社团成员本人后，点击**通过**；对可疑或非本社人员点击**拒绝**。
4. 处理后该条会从待审批列表移除；后续可在用户管理里查到其最新状态。

:::tip[找不到待审批的人？]
如果新同学说自己「注册了但列表里没有」，先确认站点是否开启了公开注册、
是否要求审批（见 [站点设置](./settings.md) 的注册开关）。若关闭了「注册需审批」，
新账号可能直接激活，不会进入审批队列。
:::

本板块对应权限码 `approvals:manage`。

## 用户管理

`/admin/users` 是全站用户列表，使用 **TanStack Table**，支持排序与筛选，
方便按角色、状态、组别等快速定位。管理员可对用户执行以下操作。

### 修改角色

在 **ADMIN（管理员）** 与 **MEMBER（成员）** 之间切换用户角色：

1. 在用户列表中找到目标用户。
2. 将其角色改为 ADMIN 或 MEMBER。
3. 改为 ADMIN 后，该用户即可进入 `/admin` 后台；改回 MEMBER 后随即失去后台权限。

:::caution
授予 ADMIN 等于把全站管理能力交给对方，请谨慎操作。角色与权限矩阵见
[角色与权限](../reference/roles-permissions.md)。
:::

### 启用 / 禁用账号

- **禁用**：账号变为 **DISABLED**，该用户**无法登录**（用于离队、误注册、临时封禁等）。
- **恢复**：把 DISABLED 账号改回 **ACTIVE**，即可重新登录。

禁用不删除数据，是可逆操作，比拒绝/删除更稳妥。

### 分配组别

给用户指定所属组别。分组后，任务可以按整组指派，无需逐人添加：

1. 在用户列表中选择用户。
2. 为其分配一个组别（组别在 [组别管理](./groups.md) 中维护）。
3. 之后在 [任务管理](./content.md) 中即可对整组下派任务，成员侧用法见
   [任务](../guide/tasks.md)。

用户管理对应权限码 `members:manage`。

## 常见处理场景

| 场景 | 建议操作 |
| --- | --- |
| 新成员注册进来 | 在审批页**通过** → 在用户管理分配**组别** |
| 误注册 / 非本社人员 | 审批页**拒绝**，或对已激活账号**禁用** |
| 成员临时离队 | **禁用**（保留数据），回归后**恢复为 ACTIVE** |
| 需要新的管理员 | 在用户管理把其角色改为 **ADMIN** |

:::note
所有审批与用户变更都会记录到 [操作日志](./audit.md)，可用于事后追责与排查。
:::
