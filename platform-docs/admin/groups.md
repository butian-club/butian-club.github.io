# 组别管理

**组别**用来把社团成员按分工（如结构组、程序组、答辩组等）组织起来。分好组后，
任务可以**按整组指派**，不必逐人添加，是比赛季高效派活的基础。

- 🧩 访问地址：[https://gfssm.butian.club/admin/groups](https://gfssm.butian.club/admin/groups)
- 本板块对应权限码 `groups:manage`。

## 组别有哪些属性

| 字段 | 说明 |
| --- | --- |
| **名称** | 组别显示名，如「结构组」 |
| **slug** | 组别的英文/短标识，用于链接与识别，建议简短、全小写 |
| **描述** | 一句话说明该组的职责范围 |
| **颜色** | 用于在界面上区分不同组别的标识色 |
| **归档（isArchived）** | 归档后组别不再用于日常指派，但历史记录保留 |

## 创建组别

1. 打开 [https://gfssm.butian.club/admin/groups](https://gfssm.butian.club/admin/groups)。
2. 点击新建组别。
3. 填写**名称**、**slug**、**描述**，并选择一个**颜色**。
4. 保存后，组别即可在用户管理与任务指派中使用。

:::tip[slug 怎么起]
slug 建议用简短、稳定的英文标识（如 `structure`、`software`、`defense`），
起好后尽量不要频繁更改，以免影响已有引用。
:::

## 编辑组别

1. 在组别列表中找到目标组别。
2. 修改名称、描述、颜色等字段并保存。
3. 修改会即时反映到成员目录、任务指派等使用该组别的地方。

## 归档组别

当某个组别不再需要（如比赛结束、分工调整）时，可将其**归档**：

- 归档（`isArchived` 为真）后，该组别不再出现在日常指派选项中。
- 历史数据（曾属于该组的成员、按组下派过的任务）**依然保留**，便于回溯。
- 相比直接删除，归档更安全、可逆。

## 把成员分到组

组别本身建好后，需要把成员分进去才能发挥作用。成员的分组在**用户管理**里完成：

1. 到 [用户管理](./approvals-users.md)（`/admin/users`）。
2. 选择用户并为其分配组别。
3. 一个成员归组后，即可被「按组指派」的任务覆盖到。

## 分组之后：整组指派任务

分好组的最大价值是**整组派活**：

1. 到 [任务管理](./content.md)（`/admin/tasks`）创建或编辑任务。
2. 指派对象选择某个**组别**，而不是逐个选人。
3. 该组的所有成员都会成为任务对象。成员侧如何查看与推进这些任务，见
   [任务](../guide/tasks.md)。

:::note
组别的增删改都会记录进 [操作日志](./audit.md)。若指派任务时找不到某个组，
先确认它是否被归档。
:::
