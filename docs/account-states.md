# 账号状态机

每个账号在任意时刻都处于四种状态之一。状态决定了这个账号**能不能登录**、
**能做什么**，以及需要管理员做哪些操作才能改变它。本页是状态流转的参考口径。

## 四种状态

| 状态 | 名称 | 一句话说明 |
| --- | --- | --- |
| `PENDING` | 待审批 | 注册后等待管理员审批，尚不能正常使用 |
| `ACTIVE` | 已激活 | 正常账号，可登录并使用其角色允许的全部功能 |
| `REJECTED` | 已拒绝 | 注册申请被管理员拒绝，无法登录 |
| `DISABLED` | 已禁用 | 曾经激活、后被管理员停用，无法登录 |

## 状态总览表

| 状态 | 可否登录 | 能做什么 | 如何进入此状态 | 如何离开 |
| --- | --- | --- | --- | --- |
| `PENDING` | 否（可登录但被拦） | 登录后被引导到 `/pending` 等待页，无法访问业务功能 | 在「注册需审批」开启时完成注册 | 管理员审批：通过 → `ACTIVE`；拒绝 → `REJECTED` |
| `ACTIVE` | 是 | 按角色（ADMIN / MEMBER）使用全部对应功能 | 审批通过；或在「注册需审批」关闭时注册即激活；或被禁用后由管理员恢复 | 管理员禁用 → `DISABLED` |
| `REJECTED` | 否 | 无（不能登录） | 管理员在审批时拒绝该 `PENDING` 申请 | 终态，需管理员另行处理 |
| `DISABLED` | 否 | 无（不能登录） | 管理员对 `ACTIVE` 账号执行禁用 | 管理员恢复 → `ACTIVE` |

:::info[「可登录但被拦」是什么意思]
`PENDING` 用户输入正确的账号密码可以通过认证，但系统会立即把他引导到等待页 `/pending`，
不允许进入任何业务模块。可以理解为「登录成功、但功能全部锁住」。
`REJECTED` 与 `DISABLED` 则连认证都不放行。
:::

## 流转规则

- **注册**：填写注册信息后——
  - 若站点设置为「注册需审批」→ 账号进入 `PENDING`，等待管理员处理；
  - 若「注册需审批」关闭 → 账号**直接** `ACTIVE`，注册后即可使用。
- **审批**（管理员在 [成员目录](https://gfssm.butian.club/members) 页面操作）：
  - `PENDING` →（通过）`ACTIVE`
  - `PENDING` →（拒绝）`REJECTED`
- **禁用 / 恢复**：
  - `ACTIVE` →（管理员禁用）`DISABLED`
  - `DISABLED` →（管理员恢复）`ACTIVE`

## 状态流转图（纯文本）

```
                    注册
                     │
        ┌────────────┴─────────────┐
        │                          │
「注册需审批」开启           「注册需审批」关闭
        │                          │
        ▼                          ▼
    ┌─────────┐                ┌─────────┐
    │ PENDING │                │ ACTIVE  │◄────────┐
    │ 待审批  │                │ 已激活  │         │
    └────┬────┘                └────┬────┘         │
         │                          │              │
   ┌─────┴──────┐            管理员禁用            管理员恢复
   │            │                  │              │
 通过         拒绝                  ▼              │
   │            │              ┌─────────┐        │
   ▼            ▼              │ DISABLED│────────┘
┌─────────┐ ┌─────────┐        │ 已禁用  │
│ ACTIVE  │ │REJECTED │        └─────────┘
│ 已激活  │ │ 已拒绝  │
└─────────┘ └─────────┘

登录判定：
  ACTIVE   → 正常进入平台
  PENDING  → 认证通过，但被引导到 /pending 等待页
  REJECTED → 不能登录
  DISABLED → 不能登录
```

## 登录行为速查

| 状态 | 登录结果 |
| --- | --- |
| `ACTIVE` | 正常登录，进入工作台 |
| `PENDING` | 认证通过但被跳转到 `/pending` 等待页，功能不可用 |
| `REJECTED` | 拒绝登录 |
| `DISABLED` | 拒绝登录 |

:::tip[相关操作在哪里]
状态的所有变更（审批、拒绝、禁用、恢复）都由管理员在后台完成，详见
[成员目录](./members.md) 的「审批与管人」。「注册需审批」这一开关属于站点设置，
决定新注册账号是落在 `PENDING` 还是 `ACTIVE`。注册流程本身见
[注册](./register.md)。
:::
