# 任务

**任务（Task）**模块是社团在比赛季分工协作的核心：谁负责什么、做到什么程度、
截止到哪天，都在这里跟进。它支持按用户或部门指派、记录进展、上传附件，
让每一件「活」都有明确的负责人和可追溯的过程。

- 📋 任务列表：[https://gfssm.butian.club/tasks](https://gfssm.butian.club/tasks)
- 任务详情：`/tasks/[id]`

:::note[权限速览]
大多数成员对任务是「查看 + 有限操作」：可以浏览任务、添加进展记录。
**创建、编辑、指派任务**属于管理动作，需要管理员权限（`tasks:manage`）。
管理员的这些操作**就在本页面（`/tasks`）内**完成——新建任务的按钮、每条任务的
编辑入口都在任务列表上，不必再单独进管理后台。详见 [角色与权限](./roles-permissions.md)。
:::

## 任务列表与筛选

打开 `/tasks` 即是任务列表。你可以按**状态**、**优先级**等条件筛选，
快速定位关心的任务。

- 筛选通过 **URL 参数 + 服务端渲染**实现——也就是说，筛选后的地址是可以直接
  复制、收藏、分享的。把带筛选参数的链接发给队友，对方打开看到的就是同一份结果。
- 常见用法：先筛出「进行中」的任务看整体进度，或筛出「高优先级」的任务优先处理。

:::tip
善用浏览器书签保存常用的筛选视图（例如「我关注的高优先级进行中任务」），
下次一键直达。
:::

## 任务字段

每个任务包含以下字段：

| 字段 | 说明 | 取值 |
| --- | --- | --- |
| **标题** | 任务的简短名称 | 文本 |
| **描述** | 任务的详细说明，支持 Markdown | Markdown 文本 |
| **状态** | 任务当前进展阶段 | `TODO` 待办 / `IN_PROGRESS` 进行中 / `DONE` 已完成 |
| **优先级** | 任务的紧急重要程度 | `HIGH` 高 / `MEDIUM` 中 / `LOW` 低 |
| **截止时间** | 期望完成的时间点 | 日期时间 |

:::info[Markdown 说明]
任务描述与进展记录都支持 GitHub 风格 Markdown（GFM，`remark-gfm`），
可以用表格、任务列表（`- [ ]` / `- [x]`）、代码块等排版。
:::

## 指派任务

任务可以指派给具体的**用户**，也可以指派给整个**部门**，适配不同的协作粒度：

- **指派给用户**：责任落到个人，适合独立完成的活。
- **指派给部门**：把任务交给一个部门共同承担，适合需要团队配合的活。
  部门的维护见 [部门](./departments.md)。
- **主负责人（isPrimary）**：在多个被指派对象中，可以标记一位**主负责人**，
  明确「这件事最终由谁兜底」，避免责任分散。

:::warning
指派、修改指派对象、设置主负责人都属于管理动作，需要管理员权限（`tasks:manage`）。
如果你是成员、发现指派有误，请联系管理员调整。
:::

## 添加进展记录

进展记录是任务模块最贴近日常协作的功能——**成员**可以在任务下持续添加**进展**，
汇报自己做到哪一步了。

1. 进入任务详情页 `/tasks/[id]`。
2. 在进展区填写你的进展说明（支持 Markdown）。
3. 提交后，进展会按时间沉淀在任务下，形成完整的过程记录。

这样负责人和管理员不用反复追问，翻一遍进展就知道整件事推进到哪了。
配合 [工作日志](./worklogs.md)，你还能从个人视角记录每天的产出。

## 附件

任务支持上传与下载**附件**，方便把设计稿、数据、参考资料等直接挂在任务上：

- 在任务详情页可以上传附件，其他有权限的成员可以下载。
- **附件访问受登录与范围限制**：必须登录、且在允许的范围内才能下载，
  不会对外公开泄露。

:::note
上传前请确认文件确实和这个任务相关；敏感资料请遵循社团的资料管理约定。
:::

## 典型流程

以一个「准备答辩 PPT」的任务为例：

1. 管理员在 `/tasks` 页面点**新建任务**，填写标题、描述、优先级（HIGH）和截止时间。
2. 管理员把任务指派给「进度成本部」，并把小美设为主负责人。
3. 小美和组员在任务下陆续添加进展：初稿完成、配图替换、终版定稿。
4. 大家把 PPT 文件作为附件上传，方便互相下载查看。
5. 完成后，管理员（或有权限者）把状态改为 `DONE`。

## 常见问题

- **我看不到「新建任务」按钮？** 创建任务需要 `tasks:manage` 权限，成员默认没有。
- **改不了任务状态 / 标题？** 编辑任务同样需要管理员权限；成员能做的是加进展。
- **下载附件提示无权限？** 附件访问受登录与范围限制，确认你已登录且在允许范围内；
  账号未激活也会受限，见 [账号状态](./account-states.md)。
- **任务被指派给我了，会有提醒吗？** 相关事件会体现在
  [通知中心](./notifications.md) 与首页动态里。
