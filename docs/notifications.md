# 通知中心

通知中心把和你相关的事件（被指派任务、工单被创建或处理、评论等）集中起来，让你不用一个页面一个页面地去翻。它是 **站内通知**：只在平台内提醒，不发邮件、不发短信。

- 📥 通知中心地址：[https://gfssm.butian.club/notifications](https://gfssm.butian.club/notifications)

## 在哪里看通知

平台顶栏有一个 **铃铛图标 🔔**，无论你在哪个页面都能看到它。

- 有 **未读通知** 时，铃铃图标上会出现 **红点**，并显示 **未读计数**。
- 点击铃铛可以快速查看最近的通知；点进 **通知中心** 页面可以看到完整列表。

:::tip[看到红点先别慌]
红点只代表「有新事件」，不一定是紧急事务。点开看一眼，处理完就可以标记为已读，把计数清零。
:::

## 系统会自动提醒哪些事

你 **不需要手动订阅**。平台会根据你在各模块里的身份，自动订阅相关事件并向你推送站内通知。常见的通知场景包括：

| 触发事件 | 谁会收到 |
| --- | --- |
| 有任务 **被指派** 给你（按用户或你所在部门） | 被指派人 |
| 工单 **被创建** | 相关处理人 / 管理员 |
| 你的工单 **被处理**（状态更新、被回复） | 工单提交人 |
| 有人在任务 / 工单等内容下 **发表评论** | 相关参与者 |

具体能收到哪些事件，取决于你的角色与权限。关于任务与工单本身怎么用，见 [任务](./tasks.md) 与 [工单](./tickets.md)。

## 标记为已读

处理完一条通知后，把它 **标记为已读**，未读计数就会相应减少，铃铛红点在没有未读时会消失。

1. 打开 [https://gfssm.butian.club/notifications](https://gfssm.butian.club/notifications)。
2. 浏览通知列表，找到已经处理或已经知晓的事件。
3. 点击 **标记为已读**，把它从未读状态移除。

:::note[已读只是状态，不会删内容]
标记已读只是把这条通知从「未读」变成「已读」，方便你清理红点。通知记录本身仍在列表里，可以回看。
:::

## 关于「没收到通知」

如果你觉得某件事没有提醒到你，请先确认以下几点：

- 平台通知是 **纯站内** 的，请回到平台查看铃铛图标，而不是等邮件或短信。
- 你能收到某类事件，取决于你是否是该事件的相关人（例如任务是否指派给你、工单是否与你有关）。
- 如果确实应该收到却一直没有，可能是权限或指派对象设置的问题，可联系管理员核实。

:::warning[不要依赖通知做唯一提醒]
通知中心是辅助提醒，不是待办清单。重要任务请同时到 [任务](./tasks.md) 页面确认截止时间与进度，避免漏掉。
:::

## 常见问题

| 问题 | 说明 |
| --- | --- |
| 会发邮件 / 短信吗？ | 不会，只有站内通知 |
| 需要自己订阅吗？ | 不需要，系统自动订阅相关事件 |
| 红点一直在怎么办？ | 打开通知中心，把已处理的通知 **标记为已读** |
| 通知会消失吗？ | 标记已读不会删除，记录仍可回看 |
