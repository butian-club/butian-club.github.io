# 操作日志

**操作日志**（审计日志）是后台的「行车记录仪」，把关键管理操作一条条记下来，
方便事后**追责**与**排查问题**——比如「这个账号是谁禁用的」「公告什么时候改的」。

- 📜 访问地址：[https://gfssm.butian.club/admin/audit](https://gfssm.butian.club/admin/audit)
- 本板块对应权限码 `audit:view`。

## 每条日志记录了什么

每条记录围绕「谁、何时、做了什么、对什么」四个要素：

| 字段 | 含义 | 例子 |
| --- | --- | --- |
| **actor（操作者）** | 执行操作的管理员 | 某管理员账号 |
| **时间** | 操作发生的时刻 | 2026-07-10 12:30 |
| **action（动作）** | 做了什么操作 | 改角色 / 发布公告 / 删除任务 |
| **target（对象）** | 操作作用的对象 | 某个用户 / 某条公告 / 某个部门 |

## 记录范围与保留策略

- **记录范围**：用户角色与状态变更、部门增删改、内容发布/置顶/归档/删除、
  站点设置与倒计时调整等**关键操作**都会被写入。
- **保留条数**：仅保留**最近约 300 条**历史；更早的记录会随新记录写入而滚动淘汰。

:::caution
由于只保留最近约 300 条，若需要长期存档，请定期**导出 CSV**（见下）留底，
不要指望日志无限期保存所有历史。
:::

## 查看与筛选

1. 打开 [https://gfssm.butian.club/admin/audit](https://gfssm.butian.club/admin/audit)。
2. 列表使用 **TanStack Table**，可按时间、操作者、动作等排序或筛选。
3. 定位到你关心的某条记录，查看其 actor、时间、action、target。

## 导出 CSV

需要留底、离线分析或上报时，可把日志**导出为 CSV**：

1. 在 [https://gfssm.butian.club/admin/audit](https://gfssm.butian.club/admin/audit) 点击导出。
2. 得到当前日志的 CSV 文件，可用 Excel / 表格软件打开。
3. 建议在重要节点（如比赛结束、管理员交接）定期导出归档。

## 典型排查场景

| 你想弄清楚 | 怎么用日志 |
| --- | --- |
| 某账号为何登不上 | 按对象筛出该用户，看是否有「禁用」记录及操作者 |
| 谁改了某条公告或设置 | 按对象或动作筛，定位 actor 与时间 |
| 某段时间发生了哪些管理动作 | 按时间排序，通读该时间段的记录 |
| 需要向社团交代管理留痕 | 导出 CSV 作为凭据 |

:::note
操作日志是**只读**的审计视图，不能在此处直接改数据。要恢复某个误操作，
请回到对应板块（如 [成员目录](./members.md)，或对应的功能页面）执行相应动作。
:::
