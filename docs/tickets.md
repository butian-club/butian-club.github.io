# 工单

**工单（Ticket）**是成员向社团「提需求、报问题」的正式通道。无论是申请资源、
反馈故障，还是提出建议，都可以开一张工单，由管理员接手处理，全程有记录、可追溯。
和任务不同，**工单通常由成员发起**，是自下而上的协作入口。

- 🎫 工单列表：[https://gfssm.butian.club/tickets](https://gfssm.butian.club/tickets)
- 新建工单：[https://gfssm.butian.club/tickets/new](https://gfssm.butian.club/tickets/new)
- 工单详情：`/tickets/[id]`

:::note[权限速览]
**成员**可以创建工单、发表评论。**处理、指派、置顶、关闭**工单属于管理动作，
需要管理员权限（`tickets:manage`）。详见 [角色与权限](./roles-permissions.md)。
:::

## 工单字段

| 字段 | 说明 |
| --- | --- |
| **标题** | 工单的简短主题 |
| **正文** | 详细描述，支持 Markdown |
| **类型** | 工单的分类，例如 `RESOURCE`（资源申请）等 |
| **状态** | 工单当前进展，例如 `OPEN`（待处理）等 |
| **是否置顶** | 置顶的工单会更显眼 |
| **指派的处理管理员** | 负责跟进该工单的管理员 |

:::info[Markdown 说明]
工单正文与评论都支持 GitHub 风格 Markdown（GFM，`remark-gfm`），
可以用表格、任务列表、代码块等，把需求或问题描述得更清楚。
:::

## 新建工单（成员）

作为成员，你可以自己开工单。步骤如下：

1. 打开 [https://gfssm.butian.club/tickets/new](https://gfssm.butian.club/tickets/new)。
2. 填写**标题**：一句话说清你要什么、遇到什么问题。
3. 选择**类型**：例如资源申请（`RESOURCE`）等，选对类型有助于管理员分流。
4. 填写**正文**：用 Markdown 把背景、需求、期望说明白。资源申请建议写清
   要什么、数量、用途、需要的时间。
5. 提交。工单进入待处理状态，等待管理员接手。

:::tip[写好工单的小建议]
- 标题具体：写「申请 2 台服务器用于集训」，而不是「求助」。
- 正文给足上下文：越清楚，管理员越能快速判断和处理，减少来回追问。
- 一张工单聚焦一件事，不同需求分开提，方便各自跟进。
:::

## 评论沟通

工单详情页支持 **Markdown 评论**，是成员与管理员来回沟通的地方：

- **成员**可以在自己或相关工单下发表评论，补充信息、回答管理员的追问。
- **管理员**用评论说明处理进度、给出结论。
- 所有评论按时间沉淀，形成完整的沟通记录。

## 管理员如何处理工单

管理员对工单拥有完整的处理权（`tickets:manage`），而且**就在工单列表页（`/tickets`）内**
操作——每条工单旁都有**处理**入口，弹出面板即可改状态、指派、置顶，不必单独进管理后台：

| 动作 | 说明 |
| --- | --- |
| **处理** | 接手工单、推进并更新状态 |
| **指派** | 把工单指派给具体的处理管理员，明确责任人 |
| **置顶** | 把重要工单置顶，让它更显眼 |
| **关闭** | 处理完成后关闭工单 |

:::warning
成员没有处理、指派、置顶、关闭工单的权限。如果你的工单迟迟没人处理，
可以在评论里 @ 提醒，或通过其他渠道联系管理员。
:::

## 工单事件与通知

工单的关键事件（例如新工单、新评论、状态变化、被指派等）会**触发通知**，
让相关的人及时知晓，不用一直盯着列表刷新。

- 你提的工单有了新回复，会通过通知提醒你。
- 管理员被指派工单时也会收到提醒。

通知的查看与管理见 [通知中心](./notifications.md)。首页也会展示工单概览动态，
见 [首页与比赛倒计时](./dashboard.md)。

## 典型流程

1. 小刚需要一批比赛用的传感器，打开 `/tickets/new` 新建工单，类型选资源申请，
   正文写清型号、数量、用途和需要的时间。
2. 系统通知管理员有新工单。
3. 管理员把工单指派给负责物资的管理员，并在评论里询问预算上限。
4. 小刚在评论里补充预算信息。
5. 管理员采购到位，更新状态并关闭工单，全过程留痕。

## 常见问题

- **我能关闭自己的工单吗？** 关闭属于管理动作，需要 `tickets:manage`；
  成员可在评论里说明「已解决，可关闭」，由管理员操作。
- **提交后没人回应？** 可在评论补充信息或联系管理员；确认账号已激活，
  见 [账号状态](./account-states.md)。
- **工单和任务有什么区别？** 工单多是成员发起的「申请 / 反馈」，
  任务多是管理员派下来的「活」。任务见 [任务](./tasks.md)。
