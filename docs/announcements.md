# 公告

**公告（Announcement）**是社团面向全体成员「广播」重要信息的方式：报名截止、
集训安排、规则变更、临时通知等，都通过公告统一发布。公告用 Markdown 编写，
可以置顶，置顶后会直接展示在首页，确保重要信息第一时间被看到。

:::note[权限速览]
**成员对公告只读**——可以阅读，但不能发布或修改。**发布和管理公告**属于管理动作，
需要管理员权限（`announcements:manage`）。详见 [角色与权限](./roles-permissions.md)。
:::

## 阅读公告

作为成员，你主要是**阅读**公告：

- 登录后，**置顶公告**会展示在 [首页](./dashboard.md)，一进来就能看到。
- 点击公告标题进入详情，阅读完整的 Markdown 正文。

:::tip
养成登录后先查看首页置顶公告的习惯，比赛季的关键节点通常都会在这里通知，
避免错过截止时间等重要信息。
:::

## 置顶与首页展示

公告可以被标记为**置顶**。置顶的作用是提升可见度：

- 置顶公告会在 [首页](./dashboard.md) 显著展示，所有成员登录即可看到。
- 适合放最重要、最紧急、需要全员知晓的通知。

:::info
置顶开关由管理员控制。重要通知过期后，管理员通常会取消置顶，
让首页始终突出当前最相关的信息。
:::

## Markdown 正文

公告正文支持 GitHub 风格 Markdown（GFM，`remark-gfm`），可以用标题、列表、
表格、任务列表、链接等排版，把通知写得条理清楚。

例如一条集训通知可以这样组织：

- **时间**：几月几日几点
- **地点**：具体位置
- **需要准备**：用任务列表 `- [ ]` 列出要带的东西
- **相关资料**：链接到对应的 [文档](./documents.md)

## 发布与管理（管理员）

发布和管理公告需要管理员权限（`announcements:manage`）。管理员可以：

| 动作 | 说明 |
| --- | --- |
| **发布公告** | 撰写 Markdown 正文并发布 |
| **置顶 / 取消置顶** | 控制公告是否在首页突出展示 |
| **管理公告** | 编辑、维护已有公告 |

发布公告的一般步骤：

1. 管理员在 [首页](./dashboard.md) 就能看到公告的**发布 / 编辑 / 删除**入口——公告管理
   直接在首页完成，不必单独进管理后台。
2. 填写标题，用 Markdown 撰写正文。
3. 决定是否置顶——重要通知建议置顶到首页。
4. 发布，全体成员即可看到；公告还会自动在聊天的 [公司总群](./chat.md) 播报一条，触达更全面。

:::warning
成员没有发布或修改公告的权限。如果你有需要全社知晓的信息，
可以联系管理员代为发布，或通过 [工单](./tickets.md) 提交你的诉求。
:::

## 写好公告的建议

- **标题点题**：让人不用打开就知道大概讲什么。
- **信息完整**：时间、地点、需要做什么、截止到几点，一次说清。
- **善用置顶**：真正重要的才置顶，置顶太多等于没有重点。
- **及时清理**：过期通知及时取消置顶，保持首页整洁。

## 常见问题

- **我看不到「发布公告」入口？** 发布 / 管理公告需要 `announcements:manage`，
  成员默认只读。
- **首页没看到公告？** 可能暂时没有置顶公告，或你的账号尚未激活，
  见 [账号状态](./account-states.md)。
- **公告和文档怎么选？** 一次性、需要全员及时看到的用公告（可置顶到首页）；
  需要长期查阅、沉淀的知识放 [文档](./documents.md)。
- **公告会触发通知吗？** 置顶公告主要靠首页展示触达；和自己相关的任务、
  工单事件提醒见 [通知中心](./notifications.md)。
