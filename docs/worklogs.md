# 工作日志

工作日志（WorkLog）用来记录你 **每天做了什么**：写了哪些代码、跑了哪些实验、开了什么会、推进到哪一步。它是你个人的工作流水账，方便自己复盘，也方便在总结时快速回忆。

- 📓 记录入口在个人中心：[https://gfssm.butian.club/profile](https://gfssm.butian.club/profile)

## 它是什么

| 特点 | 说明 |
| --- | --- |
| **归属** | 属于 **个人内容**，记录你自己的工作 |
| **正文格式** | 支持 **Markdown**，可以写标题、列表、代码块、链接等 |
| **可维护** | 每条日志都可以 **编辑** 或 **删除** |
| **查看权限** | 需要 `worklogs:view` 权限 |

:::note[工作日志 ≠ 任务]
工作日志是你主动写的「我今天做了什么」，任务是别人派给你的「你需要做什么」。两者互补：任务负责分工与跟进（见 [任务](./tasks.md)），工作日志负责沉淀你的实际投入。
:::

## 在哪里写

工作日志的记录入口就在 **个人中心（/profile）**，和个人资料、外观设置在同一个页面。关于个人中心的其他设置，见 [个人中心与外观](./profile.md)。

1. 打开 [https://gfssm.butian.club/profile](https://gfssm.butian.club/profile)。
2. 找到 **工作日志** 区域。
3. 在正文框里用 Markdown 写下今天的工作内容。
4. 保存，即新增一条日志。

:::tip[养成每天一条的习惯]
建议每天固定时间（比如收工前）花两三分钟记一条。哪怕只有一两行，日积月累也能在赛季结束时拼出完整的工作轨迹。
:::

## 写点什么

不必长篇大论，把「做了什么、进展如何、遇到什么问题」讲清楚即可。一个简单的模板：

```markdown
## 2026-07-10

- 完成了推进舱结构初稿，导出到共享文档
- 修复了倒计时组件在深色模式下的对比度问题
- 卡点：热控参数还缺一组数据，已在工单里提了申请
```

## 编辑与删除

日志写错了或需要补充，随时可以改：

| 操作 | 做法 |
| --- | --- |
| **编辑** | 打开对应日志，修改 Markdown 正文后保存 |
| **删除** | 对不再需要的日志执行删除 |

:::warning[删除通常不可撤销]
删除会移除这条日志记录，请谨慎操作。如果只是内容过时，优先考虑 **编辑** 而不是删除。
:::

## 常见问题

| 问题 | 说明 |
| --- | --- |
| 别人能看到我的工作日志吗？ | 工作日志属于个人内容，查看受 `worklogs:view` 权限约束 |
| 正文支持哪些格式？ | 支持 Markdown（标题、列表、代码块、链接等） |
| 在哪里写？ | 在 [个人中心](./profile.md)（/profile）里 |
| 打不开工作日志？ | 可能是缺少 `worklogs:view` 权限，见 [角色与权限](./roles-permissions.md) |
