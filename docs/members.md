# 成员目录

成员目录让你看到 **社团里都有谁**：他们的资料、所在部门、是管理员还是普通成员，以及此刻
是否在线。想找人对接、想知道某个头衔背后是谁，从这里开始。

- 👥 成员列表：[https://gfssm.butian.club/members](https://gfssm.butian.club/members)
- 👤 成员详情：`https://gfssm.butian.club/members/[id]`（点列表里的成员进入）

查看成员目录需要 `members:view` 权限。

:::note[AI 助手不在成员里]
平台的 AI 助手不是成员，不会出现在成员目录里——它只在 [聊天](./chat.md) 里用 `@ai` 呼叫。
:::

## 成员列表

打开 **/members** 会看到全体成员的列表。你可以在这里快速浏览，并点击任意成员进入其详情页。

1. 打开 [https://gfssm.butian.club/members](https://gfssm.butian.club/members)。
2. 浏览成员列表。
3. 点击某位成员，进入其 **详情页（/members/[id]）** 查看更多信息。

## 成员详情能看到什么

在成员详情页，你可以了解一位成员的公开信息：

| 信息 | 说明 |
| --- | --- |
| **真实姓名** | 成员填写的真实姓名 |
| **昵称** | 平台内常用的称呼 |
| **头衔** | 例如「结构部组长」「热控工程师」等 |
| **简介** | 成员自我介绍 |
| **性别展示** | 按成员自己的设置展示（不透露 / 男 / 女 / 自定义文本） |
| **所在部门** | 成员归属的部门 |
| **角色** | **管理员** 或 **成员** |
| **在线状态** | 离线 / 在线 / 空闲 / 工作中 / 忙碌 |

:::note[资料由成员自己维护]
这些资料是成员在 [个人中心](./profile.md) 里填写的。看到的性别展示形式，也由对方自己
选择（可以选择不透露）。想修改你自己的资料，去个人中心即可。
:::

## 在线状态怎么理解

在线状态像 QQ 一样反映成员当前的可联系程度：**不在线时自动显示灰色的「离线」**；在线时，
成员可以自己把状态设成下面几种（在顶栏切换，见 [个人中心的在线状态](./profile.md)）：

| 状态 | 含义 |
| --- | --- |
| **离线**（灰） | 当前不在使用平台——自动判定，无需手动设置 |
| **在线**（绿） | 正在使用平台，通常可以联系 |
| **空闲** | 在线但暂时离开一会儿 |
| **工作中** | 正在专注推进事情 |
| **忙碌**（黄） | 在线但不希望被打扰 |

:::tip[找人前先看状态]
需要即时对接时，优先找 **在线 / 空闲** 的成员；看到 **忙碌** 时，考虑用
[聊天](./chat.md) 留言或提 [工单](./tickets.md)，让对方有空再回。
:::

## 查看 vs 管理

成员目录对普通成员是 **只读** 的：你能看资料、部门、角色和状态，但不能修改别人的账号。
**管理成员的操作，管理员就在本页面内完成**（不必再进管理后台）。

| 能力 | 谁可以 | 权限 |
| --- | --- | --- |
| 浏览成员、查看详情 | 拥有查看权限的成员 | `members:view` |
| 审批账号、改角色、启停、分部门、发邀请码 | 管理员 | `members:manage` |

## 管理员：审批与管人

管理员在 [成员目录](https://gfssm.butian.club/members) 页面上，能看到普通成员看不到的管理
入口。所有「管人」的活都集中在这里：

### 审批新账号

新注册的账号处于**待审批（PENDING）**状态，不能登录，需要管理员在成员页面处理：

- **通过**：账号变为 **ACTIVE**，用户即可登录使用。
- **拒绝**：账号变为 **REJECTED**，无法登录。

确认是社团成员本人再通过；可疑或非本社人员则拒绝。账号四种状态的完整流转见
[账号状态说明](./account-states.md)。

### 改角色、启用 / 禁用

- **改角色**：在 **管理员（ADMIN）** 与 **成员（MEMBER）** 之间切换。改成 ADMIN 即拥有全站
  管理能力，请谨慎。
- **禁用**：账号变为 **DISABLED**，无法登录（用于离队、误注册、临时封禁）。**恢复**则改回
  ACTIVE。禁用不删数据，可逆，比拒绝/删除更稳妥。

### 分配部门

给成员指定所属**部门**。分好后，任务可**按整组指派**、成员自动进入对应的
[部门频道](./chat.md)。部门本身在 [部门](./departments.md) 页面维护。

### 邀请码

需要定向拉人进来时，管理员可在成员页面**生成邀请码**，让新成员凭码注册（可指定
自动加入的部门）。

:::caution
授予 ADMIN 等于把全站管理能力交给对方；禁用 / 拒绝会挡住对方登录。这些动作都会写入
[操作日志](./audit.md)，便于事后追溯。角色与权限矩阵见
[角色与权限](./roles-permissions.md)。
:::

## 常见问题

| 问题 | 说明 |
| --- | --- |
| 打不开成员目录？ | 可能缺少 `members:view` 权限 |
| 想改别人的部门 / 角色？ | 需要管理员，直接在 `/members` 页面操作 |
| 新同学注册了却进不来？ | 账号待审批，管理员在成员页面**通过**即可 |
| 为什么看不到某人性别？ | 对方可能选择了「不透露」 |
| 怎么修改我自己的资料？ | 去 [个人中心](./profile.md)（/profile） |
