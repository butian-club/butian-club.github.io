# 部门

**部门**把社团成员按分工组织起来——本届 GFSSM 决赛分为**管理层**与**结构 / 运营 / 人因 /
基建 / 市场**五个业务部门。分好部门后，任务能**按整组指派**、聊天里会**自动建部门频道**，
是比赛季协作的基础。

- 🏢 访问地址：[https://gfssm.butian.club/departments](https://gfssm.butian.club/departments)

## 看什么

部门页面把每个部门做成一张卡片，一眼看清团队结构：

- **部门名称与颜色**：颜色会在成员目录、任务、聊天等处用于区分部门。
- **职责描述**：这个部门管什么。
- **成员**：属于该部门的人，以及人数。

:::note[管理层排在最前]
所有部门里，**管理层**固定排在第一位，之后是各业务部门。管理层统筹协调、资源调度与
对外答辩节奏，其余部门各司其职。
:::

部门和成员是打通的：想看某个人的资料，去 [成员目录](./members.md)；想知道谁负责哪块，
看这里的部门划分即可。

## 部门 × 其他功能

| 部门用在哪 | 效果 |
| --- | --- |
| **任务指派** | 任务可直接指派给一个**部门**，整组成员都成为任务对象，不用逐个点，见 [任务](./tasks.md) |
| **聊天频道** | 每个部门自动有一个**部门频道**，成员自动加入，见 [聊天](./chat.md) |
| **@提及** | 聊天里 `@部门名` 可一次提醒整个部门 |
| **成员目录** | 成员按部门归类展示，见 [成员目录](./members.md) |

## 管理员：维护部门

:::tip[管理直接在本页做]
部门的增删改**就在部门页面内**完成——管理员会在页面上看到额外的操作入口，
不用再单独进管理后台。普通成员看到的只是展示。
:::

管理员（ADMIN）在 [部门页面](https://gfssm.butian.club/departments) 可以：

### 新建部门

1. 点页面上的**新建部门**。
2. 填写**名称**、**slug**（英文短标识）、**描述**，并选一个**颜色**。
3. 保存后，该部门即可用于成员分组、任务指派，聊天里也会自动出现对应频道。

:::tip[slug 怎么起]
slug 用简短、稳定的英文标识（如 `structure`、`operations`、`marketing`），
起好后尽量别频繁改，以免影响已有引用与频道。
:::

### 编辑部门

在目标部门卡片上打开**编辑**，改名称、描述、颜色后保存。改动会即时反映到成员目录、
任务指派、聊天频道等处。

### 归档部门

某个部门不再需要时，可将其**归档**，而不是直接删除：

- 归档后，该部门不再出现在日常指派与新建选项中。
- 曾属于它的成员、按它派过的任务等**历史数据仍然保留**。
- 已归档的部门会集中在页面的**归档区**，需要时可**恢复**。

相比删除，归档更安全、可逆。

### 把成员分到部门

部门建好后，要把成员分进去才生效。成员的分部门在 [成员目录](./members.md)（`/members`）里完成：
选中用户 → 指定其所属部门。分好后，该成员就会被「按部门指派」的任务覆盖到，也会自动进入
对应的部门频道。

## 常见问题

| 问题 | 说明 |
| --- | --- |
| 指派任务时找不到某部门？ | 先确认它没有被**归档** |
| 新建部门后频道没出现？ | 部门频道在有人访问聊天时自动补建，稍等或刷新聊天 |
| 成员进错部门了？ | 到 [成员目录](./members.md) 改他的所属部门 |
| 归档后数据会丢吗？ | 不会，历史保留，可随时恢复 |
